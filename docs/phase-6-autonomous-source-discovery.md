# Phase 6 - Autonomous Source Discovery

## Objective

Phase 6 turns StackMend's learning pipeline into a source-aware system that discovers new engineering knowledge and proves it before promoting it into active intelligence.

The governing loop is:

```text
Discover source
-> identify publisher
-> verify authority
-> inspect licensing
-> check freshness
-> validate examples
-> compare with independent sources
-> sandbox execution
-> assign trust score
-> approve or quarantine
-> continuously revalidate
```

Beacon is the subsystem responsible for this loop.

## Product rule

No technique enters StackMend's active intelligence unless the source is:

- attributable
- legally usable
- independently corroborated
- technically tested
- version-scoped
- continuously revalidated

## Core deliverables

- autonomous discovery missions driven by measured capability gaps
- source identity verification
- source tiering and trust scopes
- claim-level corroboration
- executable claim trials
- source licensing and automation-permission enforcement
- source drift monitoring
- quarantine workflows
- ProofChain lineage for promoted techniques

## Operating modes

- Discovery Mode
- Verification Mode
- Learning Mode
- Revalidation Mode
- Emergency Quarantine Mode

## Key outcomes

Beacon should let StackMend answer:

- Where did this technique come from?
- Why is this source trusted?
- Which exact claims were verified?
- Under which versions does this still hold?
- Should this knowledge be suspended?

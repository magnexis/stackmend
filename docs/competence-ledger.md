# Competence Ledger

## Purpose

The `Competence Ledger` records whether StackMend is actually getting better across repeated work.

It should emphasize measurable competence, not storage growth.

## Key metrics

- repeated mistake rate
- diagnostic precision
- diagnostic recall
- repair acceptance rate
- repair modification rate
- repair reversion rate
- first-attempt repair success
- confidence calibration
- time to confirmed resolution
- cross-project transfer success

## Strongest question

The most important test is:

> Does StackMend solve equivalent problems more correctly, with fewer attempts, after interacting with users?
